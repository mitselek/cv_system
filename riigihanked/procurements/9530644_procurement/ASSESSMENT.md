## Lessons Learned

### For Future Procurement Assessments

**1. Read Technical Description First**

- CPV code can be misleading (this was "software services" but actually hardware installation)
- Title can be ambiguous ("system upgrade" could be software OR hardware)
- Technical description reveals actual work type

**2. Check for Certification Requirements**

- Manufacturer certifications are hard blockers (Inner Range, Cisco, SAP, etc.)
- Cannot be obtained quickly (weeks/months of training + exams)
- If missing required certification → SKIP immediately

**3. Assess Procurement Type**

- Software development: Code, APIs, databases, web apps → YOUR DOMAIN
- Hardware installation: Physical systems, wiring, mounting → NOT YOUR DOMAIN
- Services: Consulting, training, auditing → EVALUATE CASE-BY-CASE

**4. Timeline Reality Check**

- <1 week deadline: Need to drop everything, very high stress
- <24 hours deadline: Impossible for complex procurements
- If deadline too short for proper preparation → SKIP

**5. Profile Match First**

- Before diving into details, ask: "Is this my type of work?"
- Software developer ≠ Hardware installer
- Software developer ≠ Security system integrator
- Software developer ≠ Electrical contractor
