## Questions They May Ask

### Technical Questions

**Q: "Kuidas lahendaksid jõudluse probleemi Node.js rakenduses?"**

A: "Esiteks identifitseerin kitsaskoha - kas see on CPU, mälu või I/O? Node.js puhul on tüüpiline probleemkoht sünkroonsed operatsioonid, mis blokeerivad event loop'i. PÖFF projektis optimiseerisime andmebaasi päringuid PostgreSQL-is, kasutasime connection pooling'ut ja cache'isime sagedasti küsitavaid andmeid. Elasticsearch pipeline projektis töötlesime 100,000+ kirjet 5-minutilise tsükliga, kasutades asünkroonseid töötlusvoogusid ja batch processing'ut."

**Q: "Räägi oma Spring Boot kogemusest."**

A: "Eesti Keele Instituudis (2017-2018) juhtisin Spring Boot 2.6.12 projekti tehnilist arhitektuuri. Töötasin PostgreSQL integratsiooniga, REST API disainiga ja süsteemianalüütikuna tehnilise järelevalvega. Kuigi see oli mõni aasta tagasi, Spring Boot põhimõtted on samad - dependency injection, Spring Data, auto-configuration. Olen valmis värskendama oma teadmisi uusimate versioonide osas."

**Q: "Kas sul on Svelte kogemust?"**

A: "Mul ei ole veel Svelte/SvelteKit kogemust, kuid mul on tugev JavaScript alus - 15 aastat kogemust, 9/10 oskustase, ja Vue.js raamistiku kogemus. Svelte ja Vue on mõlemad reaktiivsed komponendi raamistikud, peamine erinevus on selles, et Svelte kompileerib komponendid tõeliseks JavaScriptiks build ajal, mitte runtime'is. See on väljakutse, mille võtan rõõmuga vastu. Olen valmis pühenduma Svelte õppimisele enne töö alustamist - olen kiire õppija ja raamistikke sõltumatu mõtlemisega arhitekt."

**Q: "Kirjelda oma Docker kogemust."**

A: "Olen kasutanud Dockerit tootmises Elasticsearch pipeline projektis DigitalOceanil, kus töötlesime 100,000+ kirjet reaalajas 5-minutilise latentsusega. Lõin isoleeritud keskkonna konteinerites, mis võimaldas sõltumatut skaleerimist ja deployment'i. PÖFF projektis kasutasime cloud-native arhitektuuri AWS-is. Mul on kogemus Dockerfile'ide kirjutamisega, multi-stage build'idega ja production deployment'idega."
